#              not a plain JS object/array/buffer.
#   - All untyped payloads return: "Update message not encoded properly"

#   - update.toString() shows it's a thin wrapper:
#         update(e){ this.root.update(e) && this.notify() }
#     The real encoder/decoder lives on `params.root` (the root PropertyNode)
#   - We cannot synthesize a valid `update()` payload without a captured frame
#     from the gateway. Probe 3 (WS intercept) will give us those frames.
#
#   PRACTICAL: keep using params.write(key, value) for single-key updates.
#              update() is only useful if we want to replay a captured
#              gateway-to-client property-update frame.
#
# Q3 ANSWERED: params.operate() is NOT a write API. It's a path-navigator
#              that walks the property tree and invokes callbacks at the leaf.
#   - Signature: operate(pathString, mapCallback, primitiveCallback, arrayCallback)
#                — 4 args; last 3 must be functions
#   - Errors confirm: "t is not a function", "o is not a function" — the
#     callbacks aren't optional
#   - From the source: walks `this.root` following dotted/bracketed path syntax
#     (`foo.bar[0].baz`), descends into `i.isMap` or `i.isArray` nodes,
#     dispatches to the appropriate callback at the leaf
#   - This is the internal path-resolver used by bindings, NOT a write primitive
#
#   PRACTICAL: ignore operate() for our purposes. It's plumbing.
