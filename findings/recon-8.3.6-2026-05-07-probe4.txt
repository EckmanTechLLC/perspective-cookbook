#   The store object IS created and returned. We just couldn't pretty-print it.
#
# IMPLICATION: We can create "ghost" view stores client-side, with no server
# involvement, and use them as MobX-backed state buckets. They:
#   - Have full property tree API (params, custom, props, root)
#   - Fire subscribe callbacks on write
#   - Are NOT registered in page.views (you have to keep your own ref)
#   - Don't sync to the gateway (no gateway-side view = no sync target)
#
# Pattern:
#   var ghost = window.__client.page._newViewStore('ghost/scratch', {});
#   ghost.params.write('foo', 'bar');
#   ghost.params.read('foo'); // 'bar'
#   ghost.params.subscribe(fn); // reactive
#
# Use cases:
#   - Cross-Markdown-component shared state inside a single session
#   - Backing store for D3 charts, Monaco editor content, etc.
#   - Shadow data for client-side computations without polluting view.params
#
# CAVEAT: stores spawned this way aren't tracked in page.views, so you must
# keep a JS reference (window.__myGhostStore = ghost) or expose it some
# other way for code to find it later.
