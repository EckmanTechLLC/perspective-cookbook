# === The mountPath="C" mystery: SOLVED ===
#
# `page.views` has exactly 1 view, and that view IS our Internals_Recon view
# (the only one we created). Its mountPath is literally "C".
#
# This contradicts §5's documentation which said mountPath looked like
# `cookbook/Internals_Recon@0`. In 8.3.6, mountPath has been changed to an
# opaque short identifier (single char here, likely a base64-ish counter).
# The human-readable path moved to `resourcePath` (which the probe
# confirmed exists via `view.resourcePath: OK`, though we didn't log its
# value — useful follow-up).
#
# §5 in PERSPECTIVE_INTERNALS.md needs to be updated:
#   - mountPath is now an opaque ID; do NOT pattern-match against it
#   - To find a specific view by user-known name, use `view.resourcePath`
#     (or page.findView() which is documented to exist).
#
